# ToDoList MVP 数据库建表最终文档

## 一、文档目标

本文档用于确定 ToDoList 项目第一版（MVP）的数据库表结构最终方案。  
目标是让数据结构设计文档、接口设计文档、Java 实体类设计、数据库实际建表保持一致，方便后续直接进入开发。

本版本围绕以下核心闭环展开：

**添加任务 → 完成任务 → 获得积分 → 兑换奖励 → 查看目标进度**

---

## 二、建表设计原则

第一版数据库设计遵循以下原则：

1. **只保留 MVP 真正需要的表**
   - 不为了“以后可能会用”而提前建很多表
   - 先保证核心业务闭环能跑通

2. **字段命名统一**
   - 表名使用小写下划线风格
   - 字段名使用小写下划线风格
   - 与 Java 实体类中的驼峰命名一一对应

3. **优先适合 Java + Spring Boot + MySQL**
   - 便于 JPA / MyBatis 映射
   - 便于 DTO、Service、Controller 理解

4. **先控制复杂度**
   - 第一版不引入子任务表、兑换记录表、任务完成历史表等复杂结构
   - 后续版本再逐步拆分

---

## 三、第一版建议保留的表

第一版建议只保留以下 4 张核心表：

1. `task`：任务表
2. `reward`：奖励表
3. `goal`：目标表
4. `user_stats`：用户统计表

---

## 四、字段类型统一约定

### 4.1 主键类型

第一版建议统一使用：

- `BIGINT` 作为主键类型
- 主键自增：`AUTO_INCREMENT`

原因：
- 对 Java 项目友好
- 数据库中常见
- 比字符串 ID 更容易上手

---

### 4.2 时间字段

统一使用：

- `DATETIME`：记录具体时间
- `DATE`：记录日期

推荐约定：

- `created_at`：创建时间
- `updated_at`：更新时间
- `completed_at`：完成时间

---

### 4.3 布尔类型

使用：

- `BOOLEAN`

例如：

- `is_long_term`

---

### 4.4 枚举字段存储方式

第一版建议使用：

- `VARCHAR`

原因：
- 可读性更强
- 调试方便
- 对新手更友好
- 后期也便于直接看数据库值

例如：

- `type`
- `status`
- `time_slot`

---

## 五、task 表设计

### 5.1 表用途

`task` 表用于保存任务数据，支撑：

- 首页任务列表
- 长期任务
- 任务完成
- 时间段分类
- 目标关联

---

### 5.2 字段清单

| 字段名 | 类型 | 非空 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | 是 | 自增 | 主键 |
| title | VARCHAR(100) | 是 | 无 | 任务名称 |
| description | VARCHAR(255) | 否 | NULL | 任务说明 |
| type | VARCHAR(20) | 是 | 无 | 任务类型：ONCE / REPEAT |
| status | VARCHAR(20) | 是 | 无 | 任务状态：TODO / COMPLETED |
| points | INT | 是 | 无 | 完成任务可获得积分 |
| is_long_term | BOOLEAN | 是 | FALSE | 是否长期任务 |
| goal_id | BIGINT | 否 | NULL | 关联目标 ID |
| time_slot | VARCHAR(20) | 否 | NULL | 时间段：MORNING / AFTERNOON / EVENING |
| display_order | INT | 否 | NULL | 展示顺序 |
| created_at | DATETIME | 是 | 无 | 创建时间 |
| updated_at | DATETIME | 是 | 无 | 更新时间 |
| completed_at | DATETIME | 否 | NULL | 完成时间 |

---

### 5.3 建表 SQL

```sql
CREATE TABLE task (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(100) NOT NULL,
  description VARCHAR(255) DEFAULT NULL,
  type VARCHAR(20) NOT NULL,
  status VARCHAR(20) NOT NULL,
  points INT NOT NULL,
  is_long_term BOOLEAN NOT NULL DEFAULT FALSE,
  goal_id BIGINT DEFAULT NULL,
  time_slot VARCHAR(20) DEFAULT NULL,
  display_order INT DEFAULT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  completed_at DATETIME DEFAULT NULL
);
```

---

### 5.4 字段约束建议

#### `title`
- 不能为空
- 长度建议不超过 100

#### `points`
- 必须大于等于 0

#### `type`
建议限制为：
- `ONCE`
- `REPEAT`

#### `status`
建议限制为：
- `TODO`
- `COMPLETED`

#### `time_slot`
建议限制为：
- `MORNING`
- `AFTERNOON`
- `EVENING`

## 九、表关系说明

第一版表关系尽量保持简单。

### 9.1 task 与 goal

关系：
- 一个目标可以关联多个任务
- 一个任务最多关联一个目标

即：

- `goal 1 ---- N task`

关联字段：

- `task.goal_id -> goal.id`

第一版可以先不强制建立外键，等你熟悉之后再补。

---

### 9.2 user_stats 与其他表

第一版中：

- `task` 不直接存用户积分结果
- `reward` 不直接存用户信息
- 用户的积分和统计统一保存在 `user_stats`

如果后续加入登录系统，再逐步把 `user_id` 补充到任务、奖励记录等表里。

## 十、推荐索引设计

第一版虽然数据量不会大，但建议先加少量有价值的索引。

### 10.1 task 表索引

建议添加：

- `idx_task_status`
- `idx_task_goal_id`
- `idx_task_time_slot`

SQL 示例：

```sql
CREATE INDEX idx_task_status ON task(status);
CREATE INDEX idx_task_goal_id ON task(goal_id);
CREATE INDEX idx_task_time_slot ON task(time_slot);
```

---

### 10.2 reward 表索引

建议添加：

- `idx_reward_status`

SQL 示例：

```sql
CREATE INDEX idx_reward_status ON reward(status);
```

---

### 10.3 goal 表索引

如果第一版只有少量目标，这里可以暂时不加索引。  
后续如果目标变多，可考虑：

```sql
CREATE INDEX idx_goal_status ON goal(status);
```

## 十四、最终推荐 SQL 汇总版

```sql
CREATE TABLE goal (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(100) NOT NULL,
  description VARCHAR(255) DEFAULT NULL,
  progress INT NOT NULL,
  status VARCHAR(20) NOT NULL,
  start_date DATE DEFAULT NULL,
  end_date DATE DEFAULT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL
);

CREATE TABLE task (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(100) NOT NULL,
  description VARCHAR(255) DEFAULT NULL,
  type VARCHAR(20) NOT NULL,
  status VARCHAR(20) NOT NULL,
  points INT NOT NULL,
  is_long_term BOOLEAN NOT NULL DEFAULT FALSE,
  goal_id BIGINT DEFAULT NULL,
  time_slot VARCHAR(20) DEFAULT NULL,
  display_order INT DEFAULT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  completed_at DATETIME DEFAULT NULL
);

CREATE TABLE reward (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(100) NOT NULL,
  description VARCHAR(255) DEFAULT NULL,
  cost_points INT NOT NULL,
  status VARCHAR(20) NOT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL
);

CREATE TABLE user_stats (
  user_id BIGINT PRIMARY KEY,
  total_points INT NOT NULL,
  total_completed_tasks INT NOT NULL DEFAULT 0,
  updated_at DATETIME NOT NULL
);

CREATE INDEX idx_task_status ON task(status);
CREATE INDEX idx_task_goal_id ON task(goal_id);
CREATE INDEX idx_task_time_slot ON task(time_slot);
CREATE INDEX idx_reward_status ON reward(status);
```
